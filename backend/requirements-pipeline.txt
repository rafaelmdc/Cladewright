# Deps to BUILD a game-data asset (manage.py build_gamedata --enrich braidworks).
# Serving deps + real enrichment via Braidworks: the wikidata weaver (common names +
# enwiki title + sitelink count) and the wikipedia weaver (pageviews) that together drive
# the "fame" popularity ranking.
#
# Braidworks is installed straight from its public GitHub repo, pinned to an immutable
# tag, so the pipeline worker image is the SAME in test and prod (no vendored wheels to
# drift). braidworks-core is the weavers' dependency, pinned to the same ref for a
# reproducible build. Needs `git` in the image (see Dockerfile.pipeline). To bump: tag a
# new release in rafaelmdc/braidworks and change the @ref on all three lines.
-r requirements.txt
braidworks-core  @ git+https://github.com/rafaelmdc/braidworks@cladewright-weavers-v0.3.0#subdirectory=braidworks-core
wikidata_weaver  @ git+https://github.com/rafaelmdc/braidworks@cladewright-weavers-v0.3.0#subdirectory=weavers/wikidata_weaver
wikipedia_weaver @ git+https://github.com/rafaelmdc/braidworks@cladewright-weavers-v0.3.0#subdirectory=weavers/wikipedia_weaver
